Add Agent

Set up a KITT agent on a GPU server

1. Install

Run this on the GPU server. It creates a virtual environment, downloads the agent package, and configures it to connect to this server.

curl -fL {{ server_url }}/api/v1/agent/install.sh | bash

2. Start

Start the agent in the foreground:

~/.kitt/agent-venv/bin/kitt-agent start

Or install as a systemd service for persistent deployment:

~/.kitt/agent-venv/bin/kitt-agent service install

3. Verify

Once the agent starts, it will appear on the Agents page within a few seconds.

Configuration

The install script automatically configures the agent with this server's URL{% if auth_token %} and auth token{% endif %}. The config is stored at ~/.kitt/agent.yaml:

server_url: {{ server_url }}
token: {{ auth_token or '(empty â€” server has no auth token)' }}

To reconfigure an existing agent:

~/.kitt/agent-venv/bin/kitt-agent init --server {{ server_url }}{% if auth_token %} --token <TOKEN>{% endif %}

Manual install

If you prefer to install step by step:

# Create virtual environment
python3 -m venv ~/.kitt/agent-venv

# Download and install the agent package
curl -fL {{ server_url }}/api/v1/agent/package -o /tmp/kitt-agent.tar.gz
~/.kitt/agent-venv/bin/pip install /tmp/kitt-agent.tar.gz

# Configure the agent
~/.kitt/agent-venv/bin/kitt-agent init --server {{ server_url }}{% if auth_token %} --token {{ auth_token }}{% endif %}

# Start the agent
~/.kitt/agent-venv/bin/kitt-agent start
